User research tracking tool for behavioral analysis

Follow the tracks of your user.

The tool is an utility to track the logs during a usability test session, in order to quickly evaluate the interaction and design concepts before going to production. Get your hands on with the tutorial and find out you how to use the tool in your UX prototypes.

Tutorial Follow the next 6 steps to go through the example.

Step 1 \\ Set up a development environment

Open the SAP Cloud Platform Cockpit and log on.
  • Navigate to the "Europe (Rot) - Trial" Region, and select your personal trial account.
  • From the left Side Navigation select the "Applications" and go to the "HTML5 Aplications".
  • Create a "New Application" by giving a name to your application and "Save" it.
  • On the right side of the list with the applications, among the actions, find the one regarding editing in SAP Web IDE. By clicking on it, a new tab opens with the editor.
  • You will be asked to clone the Git Repository of your new application, but you do not need to do it for this exercise.

Step 2 \\ Have an instance of a prototype running

Open the following Github repository that we prepared for you and download it as a ZIP.
  • Now, back to the SAP Web IDE, go to the top menu and select "File" > "Import" > "From File System".
  • Browse your files and select the ZIP file you previously downloaded and select "OK".
  • On the left side, in the tree navigation, you will see then the file structure of the project being updated.
  • Try to run it by clicking on the "Play" button in the action toolbar on the top.
  • Congratulations, you have now a working UI5 prototype as a project in your workspace.

Step 3 \\ Use case data structure

Open the UseCaseData.json file that contains the use case related information structured hierarchically.
  • In the example, in order to keep it simple, we have actually a single use case with two actions.
  • The number of use cases, actions and steps can be edited according to the needs of the usability testing.
  • The use case has an ID and a title that can be edited.
  • The action have also an ID, a title and a description that can be edited.
  • Each action contains further steps. Tehse steps have also an ID, a title and a description that can be edited.

Step 4 \\ Learn by example

The two actions have been already implemented for you. Let's go thoroughly through the first one.
  • Under the controller folder open the rootApp.controller.js file.
  • Search for UxBeginUseCase in the file and you will land in the first call of the API for beginning the first use case. That takes places on the initial loading of the page.
  • The complete syntax is the following:

    UxBeginUseCase(0, 'App initial load');
  • Open now the Home.controller.js file under the same folder.
  • Search for UxCompleteStep in the file and you will land at the point where the first step of the first action is completed. That is by clicking on the 'Goals Management' cell.
  • The complete syntax is the following:

    UxCompleteStep(0, 0, 0, 'Goals Management cell');
  • Open now the GoalsManagement.controller.js file under the same folder.
  • Search again for UxcompleteStep in the file and you will land at the point where the second step of the first action gets completed. That is by clicking on the radio button with the API key.
  • The complete syntax is the following:

    UxCompleteStep(0, 0, 1, 'Radio button for API key');
  • In the same file search for the next UxCompleteStep and you will land at the point where the third step of the first action gets completed. That is by clicking on the "Subscribe" button.
  • The complete syntax is the following:

    UxCompleteStep(0, 0, 2, 'Subscribe button');
  • Simultaneously, the first action is also completed and the next one begins automatically.
  • Under the controller folder open again the rootApp.controller.js file.
  • This time search for UxCompleteStep in the file and you will land at the point where the first step of the second action is completed. That is by clicking on the menu item on the left for the catalog.
  • Open now the Catalog.controller.js file under the same folder.
  • The complete syntax is the following:

    UxCompleteStep(0, 1, 0, 'Side navigation - Catalog item');
  • Search for UxCompleteStep in the file and you will land at the point where the second step of the second action is completed. That is by clicking on the 'Talent Management' list item.
  • The complete syntax is the following:

    UxCompleteStep(0, 1, 1, 'Master list item for Talent Management API');
  • Open now the TalentManagement.controller.js file under the same folder.
  • Search for UxCompleteStep in the file and you will land at the point where the third step of the second action is completed. That is by clicking on the radio button with the API key.
  • The complete syntax is the following:

    UxCompleteStep(0, 1, 2, 'Radio button for API key');
  • In the same file search for the next UxCompleteStep and you will land at the point where the fourth step of the second action gets completed. That is by clicking on the "Subscribe" button.
  • At this stage of the tutorial, the second action has been also completed, and therefore the use case ends also autmatically. The tool will generate a CSV file, which will be directly downloaded.

Step 5 \\ Test your prototype

Refresh the browser tab where you ran your prototype and open the browser's console.
  • Go thoroughly through all steps of the use case.
  • Make sure that on the completion of the steps a CSV file was downloaded.
  • Open your results in Microsoft Excel.

Step 6 \\ View and manage your test results over a dashboard

Check the beta version of a dashboard designed as the proper tool for reviewing the results of the usability test sessions with the tracking tool.
